## Обзор административных функций

Как администратор, вы имеете доступ к расширенным функциям системы:

- **Управление документами** - загрузка, организация, контроль доступа
- **Инициализация RAG** - настройка ИИ-ассистента для отделов
- **Мониторинг использования** - статистика и аналитика
- **Управление пользователями** - права доступа и роли
- **Техническое обслуживание** - обновления и оптимизация

## Управление пользователями

### Уровни доступа

Система поддерживает несколько уровней доступа:

1. **Уровень 1** - Базовый доступ (стажеры, временные сотрудники)
2. **Уровень 2** - Стандартный доступ (основные сотрудники)
3. **Уровень 3** - Расширенный доступ (старшие сотрудники)
4. **Уровень 4** - Полный доступ (руководители, администраторы)

### Управление отделами

Каждый пользователь привязан к отделу, что определяет:
- Доступные документы
- Возможности ИИ-ассистента
- Результаты поиска
- Тесты и анкеты

## Управление документами

### Организация файловой структуры

Рекомендуемая структура папок:
```
ContentForDepartment/
├── 1/ (HR отдел)
│   ├── policies/
│   ├── procedures/
│   └── forms/
├── 2/ (IT отдел)
│   ├── manuals/
│   ├── guides/
│   └── policies/
└── 3/ (Финансы)
    ├── regulations/
    ├── procedures/
    └── reports/
```

### Загрузка документов

**Пошаговый процесс:**

1. **Подготовка документов**
   - Проверьте формат файлов (PDF, DOCX, TXT)
   - Убедитесь в корректности названий
   - Проверьте размер файлов (до 50 МБ)

2. **Загрузка через интерфейс**
   - Перейдите в "Администрирование" → "Загрузка файлов"
   - Выберите уровень доступа
   - Укажите отдел
   - Выберите файлы
   - Нажмите "Загрузить"

3. **Проверка загрузки**
   - Используйте "Просмотр файлов" для проверки
   - Убедитесь в корректности метаданных

### Контроль качества документов

**Рекомендации:**
- Используйте понятные названия файлов
- Добавляйте описания к документам
- Регулярно обновляйте устаревшие документы
- Удаляйте дублирующиеся файлы

## Настройка RAG системы

### Инициализация для нового отдела

**Когда нужна инициализация:**
- Добавлены новые документы в отдел
- Создан новый отдел
- Изменилась структура документов
- Пользователи жалуются на неточные ответы ИИ

**Процесс инициализации:**

1. **Подготовка**
   - Убедитесь, что все документы отдела загружены
   - Проверьте корректность метаданных
   - Уведомите пользователей о временной недоступности

2. **Запуск инициализации**
   - Перейдите в "Инициализация RAG"
   - Выберите отдел
   - При необходимости включите "Принудительная перезагрузка"
   - Подтвердите операцию

3. **Мониторинг процесса**
   - Процесс может занять от 5 минут до нескольких часов
   - Используйте "Проверить статус" для контроля
   - Не запускайте повторную инициализацию до завершения

4. **Проверка результата**
   - Убедитесь, что статус показывает "Инициализирована: Да"
   - Проверьте количество обработанных документов
   - Протестируйте работу ИИ-ассистента

### Оптимизация производительности RAG

**Рекомендации:**
- Регулярно обновляйте индексы (раз в неделю)
- Удаляйте устаревшие документы перед реиндексацией
- Мониторьте размер векторной базы данных
- Используйте принудительную перезагрузку при значительных изменениях

### Устранение проблем RAG

**Частые проблемы:**

**Проблема**: ИИ не находит информацию из новых документов
**Решение**: Запустите реиндексацию с принудительной перезагрузкой

**Проблема**: Медленные ответы ИИ
**Решение**: 
- Проверьте размер векторной базы
- Оптимизируйте количество документов
- Рассмотрите архивирование старых файлов

**Проблема**: Неточные ответы
**Решение**:
- Проверьте качество исходных документов
- Убедитесь в корректности метаданных
- Рассмотрите разбиение больших документов
